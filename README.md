# KMP WebSocket App

Мультиплатформенное клиент-серверное приложение для обмена сообщениями в реальном времени через WebSocket. Реализовано на Kotlin Multiplatform с использованием Compose Desktop для графического интерфейса и Ktor для сетевого взаимодействия.

## Архитектура

Проект разделён на три модуля:

- **`shared`** – общая логика и модель данных (`Message`), сериализация (kotlinx.serialization).
- **`server`** – серверное приложение с графическим интерфейсом (Compose Desktop). Принимает WebSocket-соединения, отображает список подключённых клиентов и полученных сообщений.
- **`client`** – клиентское приложение с графическим интерфейсом (Compose Desktop). Подключается к серверу по IP и порту, отправляет сообщения, отображает статус соединения и лог событий.

## Используемые технологии

- **Kotlin Multiplatform** – общий код для JVM.
- **Ktor** – серверная и клиентская части WebSocket.
- **kotlinx.coroutines** – асинхронность.
- **kotlinx.serialization** – сериализация/десериализация JSON.
- **Compose Multiplatform (Desktop)** – графический интерфейс.
- **Gradle Kotlin DSL** – система сборки.

## ✨ Дополнительные возможности (сверх ТЗ)

- **Сохранение истории сообщений** – сервер автоматически сохраняет все полученные сообщения в файл `messages.jsonl` (JSON Lines) и загружает их при каждом запуске.
- **Кнопка «Clear History»** – на панели управления сервера добавлена кнопка для очистки истории (активна при наличии сообщений).
- **Автоматическое переподключение клиента** – при потере соединения клиент пытается переподключиться до 5 раз с экспоненциальной задержкой, отображая текущую попытку.
- **Логирование событий** – на сервере выводится лог подключений, отключений, ошибок и действий.
- **Индикация числа клиентов** – в реальном времени показывает количество подключённых клиентов.

## Требования к окружению

- **Java 11** или выше (рекомендуется JDK 17+).
- **Gradle** (используется wrapper, отдельная установка не требуется).



## Сборка и запуск


Запуск в IntelliJ IDEA

Откройте проект, дождитесь синхронизации Gradle.

Создайте конфигурации Application:

Сервер: main class com.example.kmpwebsocketapp.MainKt, module server.

Клиент: main class com.example.kmpwebsocketapp.MainKt, module client.

Запустите сервер, затем клиент (можно несколько экземпляров, разрешив параллельный запуск).

Примечания
Для подключения с другого устройства в локальной сети укажите реальный IP-адрес сервера (не localhost) и откройте порт 8080 в брандмауэре.

История сообщений сохраняется в файл messages.jsonl в рабочей директории сервера.

## Инструкция по запуску через терминал

# Сервер
./gradlew :server:run
# Клиент (в отдельном окне)
./gradlew :client:run
